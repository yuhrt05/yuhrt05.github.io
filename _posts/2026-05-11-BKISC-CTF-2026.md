### Solution

Bài cung cấp file memdump, như thường lệ thì mình sẽ bắt đầu với pslist/pstree để xem cây tiếng trình đáng nghi

![image](/assets/images13/2.png)

Duyệt qua một loạt thì không thấy tiến trình nào đáng ngờ, nhưng lại thấy được khá nhiều liên quan đến Microsoft edge làm mình liền liên tưởng đến 1 bài viết mình có đọc được cũng trong tầm đầu tháng 5 về việc edge lưu trữ `credential` ở dạng `clear-text` trực tiếp ở trong Ram. Bài viết tại [đây](https://www.facebook.com/groups/fluttervn/permalink/2206697133200046/)

Từ giả thuyết đó, mình tìm tới tool [EdgeSavedPasswordsDumper](https://github.com/L1v1ng0ffTh3L4N/EdgeSavedPasswordsDumper?utm_source=chatgpt.com) để hiểu cách Edge lưu dữ liệu credential trong memory. Tool này hoạt động bằng cách đọc memory của tiến trình msedge.exe rồi regex hunt các chuỗi có dạng:

`https <username> <password>`

Điều này cho thấy credential của Edge thực sự tồn tại dưới dạng clear-text trong RAM. Vì challenge cung cấp full memory dump (chall.dmp), mình suy đoán rất có thể các artifact liên quan cũng đang tồn tại trực tiếp dưới dạng plaintext trong này.

Đồng thời trong quá trình phân tích mình có phân tích thêm history của edge thì thấy 1 đường link pastebin, yêu cầu password để mở. khả năng cao flag sẽ có ở đây

![image](/assets/images13/33.png)

Sau những manh mối có được, mình liền nhờ AI gen 1 đoạn script python với logic regex như trên để tìm kiếm password liên quan đến pastebin

```python
import re

# Khởi tạo biểu thức chính quy quét định dạng bộ nhớ của Edge
credential_pattern = re.compile(br'https? \s*([a-zA-Z0-9_\.\-]+)\s+([a-zA-Z0-9!@#\$%\^&\*\(\)\-\+=\[\]\{\};:,<\>\?\/]+)')

dump_path = "chall.dmp"

print("[*] Đang quét dữ liệu nhị phân trên bộ nhớ dump...")
with open(dump_path, "rb") as f:
    # Đọc luồng dữ liệu theo từng block lớn để tiết kiệm RAM
    while (chunk := f.read(1024 * 1024 * 64)):  # 64MB per block
        # Tìm kiếm liên kết Pastebin khả nghi
        if b"pastebin.com" in chunk:
            urls = re.findall(br'https?://pastebin\.com/[A-Za-z0-9]+', chunk)
            for url in urls:
                print(f"[+] Phát hiện link Pastebin: {url.decode('utf-8', errors='ignore')}")
        
        # Tìm kiếm định dạng tài khoản/mật khẩu dạng clear-text
        matches = credential_pattern.findall(chunk)
        for user, passwd in matches:
            try:
                u_str = user.decode('utf-8')
                p_str = passwd.decode('utf-8')
                print(f"[+] Khả nghi Credential -> User: {u_str} | Pass: {p_str}")
            except:
                continue
```

Script PowerShell trên có nhiệm vụ tạo và import một file `.reg` nhằm chỉnh sửa `Registry` của `Windows`. Các thay đổi chủ yếu tập trung vào cấu hình `Outlook WebView` và giảm mức bảo mật của `Internet Explorer Trusted Zone`.

Cụ thể, script cấu hình Outlook tự động tải nội dung từ địa chỉ:

`http://192.168.1.189:8386/plugin/search/`

mỗi khi người dùng mở Inbox. Đồng thời, script cũng thay đổi các thiết lập bảo mật liên quan tới `ActiveX` và `Trusted Sites` nhằm cho phép thực thi nội dung web với ít hạn chế hơn. Sau khi import registry thành công, file .reg tạm thời sẽ bị xóa để giảm dấu vết trên hệ thống.

Đến đây các artifact từ log gần như đã đủ để xác định hướng tấn công, vì vậy mình chuyển sang phân tích file `.pcap` và filter riêng traffic liên quan tới IP `192.168.1.189`
